블로그 파이프라인 앞단(주제 결정 + 초안 + 검수)을 한 번에 돌리는 메타 스킬. 주제 인자가 있으면 그대로 사용하고, 없으면 `/blog-research` 의 0단계 주제 탐색이 후보를 제시한다. 카드 이미지는 메타 블록 확인 후 별도로 `/blog-cards` 를 호출한다.

> **권장 모델:** Sonnet 4.6. 글쓰기 품질은 Opus와 큰 차이 없고 비용·속도가 유리함. 현재 모델이 Opus면 `/model sonnet` 으로 전환 후 진행 권장.

## 사용법

```
/blog                              # 주제 탐색부터 시작
/blog uv 패키지 매니저              # 주제 직접 지정
/blog --profile hmkim
/blog uv --profile hmkim
```

## 실행 흐름

### 1단계: 초안 생성 (blog-research 위임)

`blog-research` 스킬의 전체 흐름을 그대로 수행한다:

1. CLAUDE.md 규칙 및 `blog-research` SKILL.md 규칙을 적용
2. **주제 탐색** (`scripts/discover-topics.py` → 후보 10개 → 사용자 선택) → 주제 확정
3. 세션 ID 생성 → 프로파일 로드 → AI 쿼리 제안 → 사용자 승인 → Tavily 검색 (≤6 쿼리)
4. 심화 키워드 1~2회 (선택, 총 Tavily 호출 ≤8)
5. **검색 맞춤 Q&A** 를 한 번에 출력하고 사용자 답변 대기 (검색 스킵 시 고정 9문 Q&A 로 fallback)
6. 답변을 받으면 선택 섹션 포함 여부 판단 (검색 경로는 카테고리 단계에서 이미 결정, 스킵 경로는 기본 1-a 로직)
7. `posts/YYYY-MM-DD-<슬러그>.md` 에 초안 + 인라인 출처 + `## 이미지 메타` 블록 작성

### 2단계: 자동 검수/교정 (blog-review 위임)

1단계에서 생성된 파일에 대해 **자동으로** `blog-review --fix` 를 실행:

1. `blog-review` SKILL.md의 결정론 체크 A와 LLM 판단 B를 모두 수행
2. 안전한 치환(음슴체→평서체, 이모지 제거, `## 마치며` 강제, "오늘 배운 것:" 제거)은 자동 적용
3. 원본은 `<파일>.bak` 백업
4. 리포트를 사용자에게 출력

### 3단계: 다음 단계 안내

검수 리포트 끝에 다음 안내를 추가 출력:

```
---
다음 단계:
- 초안과 `## 이미지 메타` 블록을 확인/수정하세요: posts/<파일>
- 카드 이미지 생성 준비 완료되면: /blog-cards
- 특정 블록만 다시 만들기: /blog-cards <파일> --only <thumb|section-1|section-2>
```

## 규칙

- **카드 이미지 생성은 자동 실행하지 않는다.** 메타 블록을 사용자가 먼저 검수·수정할 수 있게 분리.
- 초안이 생성된 직후 사용자 확인 없이 바로 검수/교정으로 넘어간다. 검수는 비용이 없고, `--fix`가 안전한 치환만 수행하므로 의도치 않은 손실이 없다.
- 검수 결과에 **[FAIL]** 이 있으면 리포트 상단에 눈에 띄게 표시해 사용자가 수동 수정하도록 유도.
- 1단계 실행 도중 질문 출력과 2단계 자동 전환이 혼동되지 않도록, 초안 파일 경로를 2단계 시작 전에 명확히 밝힌다.
- Tavily 키(`TAVILY_API_KEY`)가 `.env` 에 없으면 1단계의 `blog-research` 가 초기에 중단한다. 이때 2단계로 넘어가지 않는다.

## 에러 대응

- 1단계에서 사용자가 질문에 답하지 않고 중단하면 2단계로 진행하지 않음
- 2단계에서 파일을 못 찾으면 1단계 경로 확인 후 재시도
- 2단계의 LLM 판단 항목은 실패해도 전체 흐름을 막지 않음 (결정론 체크 결과는 반드시 보고)
