# 블로그 글쓰기 규칙

## 대상 독자
- 개발 입문자 (코딩 경험 6개월~1년)
- 전문 용어는 반드시 한 줄 설명 추가
- 비유와 예시를 적극 활용

## 제목 구조
- 대제목(H1): 주제 전체를 아우르는 한 문장
- 소제목(H2): 각 섹션 제목
- 소소제목(H3): 섹션 내 세부 항목
- 예시:
  ```
  # Docker란 무엇인가
  ## 기술 설명
  ### 컨테이너가 뭔가
  ### 왜 써야 하나
  ## 개발 경험 공유
  ### 처음 Docker를 접했을 때
  ### 삽질했던 순간
  ## 마치며
  ```

## 글 구조 (2-섹션 형식)

### 섹션 1: 기술 설명
- 개념 → 왜 필요한가 → 어떻게 동작하나 순서
- 코드 예시 있으면 주석 필수
- 소제목은 질문형으로 ("왜 이걸 써야 하나")

### 섹션 2: 개발 경험 공유
- 1인칭으로 작성 ("나는 ~했다", "처음엔 ~인 줄 알았는데")
- 막혔던 부분, 삽질한 것, 깨달은 것 포함
- 독자가 같은 실수를 피할 수 있게

### 마치며
- 3줄 이내 요약
- "오늘 배운 것: ~" 형식
- 다음 단계 한 가지 제안

## 문체
- 평서체 (~했다, ~이다, ~한다)
- 짧은 문장 (한 문장 40자 이내 권장)
- 어려운 단어 대신 쉬운 단어
- 이모지 사용 안 함

## 파일명 규칙
- 형식: `YYYY-MM-DD-주제-슬러그.md`
- 예: `2026-04-19-docker-입문.md`

## 분량
- 총 1,500자 내외 (마크다운 기준)
- 섹션 1: 700자, 섹션 2: 600자, 마치며: 200자
