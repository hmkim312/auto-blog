블로그 파이프라인. 영문 자료를 LLM 한 번에 한국어 글로 옮기면 어순·동사 결합·명사화가 영어식으로 새어 나오는 한계가 있다. 그래서 **번역과 작문을 분리한다**. 메인 Claude 가 영문 Tavily 결과를 한국어 사실 카드로 옮기고, Codex 는 한국어 카드만 보고 글을 쓴다. 검수도 Codex 가 자가 점검한다.

> **권장 모델 (메인):** Opus 4.7. 메인 Claude 가 영문 → 한국어 사실 카드 번역까지 직접 하므로 한국어 품질 우선. Codex sub-agent 는 자체 모델 (GPT-5.5).

## 사용법

```
/blog                              # 주제 탐색부터 시작
/blog uv 패키지 매니저              # 주제 직접 지정
/blog --profile hmkim
/blog uv --profile hmkim
```

## 실행 흐름

### 1단계: 주제·키워드·검색·Q&A (메인 Claude, blog-research 위임)

`blog-research` 스킬 흐름 그대로 따른다:

1. CLAUDE.md 규칙 + `blog-research` SKILL.md 규칙 적용
2. **주제 탐색** (`scripts/discover-topics.py` → 후보 10 개 → 사용자 선택)
3. **핵심 키워드·슬러그 결정** (0.5단계) — 후보 3~5 개 추천 → 사용자 선택 → 슬러그 → 파일명 확정
4. 세션 ID → 프로파일 로드 → AI 쿼리 제안 → 사용자 승인 → Tavily 검색 (≤6 쿼리)
5. 심화 키워드 1~2 회 (선택, 총 Tavily 호출 ≤8)
6. **검색 맞춤 Q&A** 한 번에 출력 → 사용자 답변 대기

### 1.5단계: 영문 → 한국어 사실 카드 변환 (메인 Claude, NEW)

영문 Tavily 결과를 메인 Claude 가 직접 한국어 사실 카드로 옮긴다. 이 단계는 **번역만** 한다 (글 작성 X). Codex 가 영어 어순·구문에 노출되지 않게 차단하는 핵심 단계.

#### 변환 규칙

- **사실·수치·인용만** 옮긴다. 의견·해석·맥락 추가 X.
- 한 사실 = 한 카드. 카드 1 개 = 한국어 한 문장 + 출처 URL.
- 영어 단어·관용구 그대로 옮기지 않는다. 한국어 어순으로 재작성.
- 영어 수동태 → 한국어 능동형 (CLAUDE.md `## 글쓰기 핵심 원칙`).
  - `is reported` → `보고했다`, `짚었다`
  - `has been moved` → `옮겨졌다` X → `옮겼다`
- 영어식 명사구 (`reliability problems`) → 자연 한국어 (`신뢰성 문제` 가 아니라 풀어 쓴 동사형 — `결과가 흔들리는 문제`)

#### 출력 포맷

```markdown
## 한국어 사실 카드

### 카드 1: 정의·등장 배경
- 사실: 하네스 엔지니어링은 모델 둘레의 도구·권한·피드백 루프를 짜는 분야다.
- 시점: 2026년 3월부터 해외 AI 블로그·뉴스레터에서 표현이 자주 보였다.
- 출처: nxcode.io/...

### 카드 2: 핵심 사례
- 사실: 한 코딩 에이전트가 모델을 그대로 두고 하네스만 다듬어 터미널 벤치 2.0 에서 30위에서 5위로 올라갔다.
- 시점: 2026년 langchain 발표 시점 기준.
- 출처: langchain.com/blog/...

### 카드 N: ...
```

카드 개수 제한 X. Tavily 결과 중 본문에 들어갈 사실만 추려 5~10 개. 사용자 답변 (Q&A) 도 같은 형식으로 함께 옮긴다 (1인칭 보존).

이 카드 묶음이 2-A 작성자에게 들어가는 **유일한 사실 입력**이다.

### 2단계: 본문 작성 + 자가 검수 (Codex)

#### 2-A. Codex 작성 (한국어 카드만 입력)

`Agent` 툴로 `codex:codex-rescue` sub-agent 호출. 메인 Claude 가 다음을 프롬프트에 넣는다 (Codex 가 cat 으로 다시 읽지 않게):

- CLAUDE.md 핵심 룰 발췌 (글쓰기 핵심 원칙·골든 톤 표본·제목 구조·도입부·문체·톤 가드·영어 용어·SEO)
- 사용자 메모리 발췌 (자연 한국어·구어 회피·메타 멘트 금지·번역 작성 룰·헤딩 종결 등)
- internal-links.md ILJ 키워드 표
- 1.5단계의 한국어 사실 카드 묶음
- 사용자 Q&A 답변 (한국어 카드와 함께)

**영문 Tavily 결과 원문은 절대 넣지 않는다.** Codex 가 영어 어순에 다시 노출되면 1.5단계가 무의미해진다.

프롬프트 골격:

```
너는 한국어 블로그 작성자다. 아래 한국어 사실 카드와 사용자 답변만 보고 글을 쓴다.
영어 자료는 받지 못했다. 한국어 카드 안의 사실·수치·인용만 활용한다.

## 작성 룰 (CLAUDE.md 발췌)
<핵심 원칙·골든 톤 표본·제목 구조·도입부·문체·톤 가드·영어 용어·SEO 본문>

## 자연 한국어 판단 기준 (사용자 메모리)
<자연_한국어·번역_작성·구어_회피·메타_멘트_금지·헤딩_종결·H2_도입_금지·도구_윤색·음역_표기 본문>

## ILJ 키워드 표
<internal-links.md 표>

## 글 정보
- 주제: <주제>
- 핵심 키워드 (H1·도입·alt 일관): <0.5단계 키워드>
- 파일 경로: posts/YYYY-MM-DD-<슬러그>.md

## 한국어 사실 카드
<1.5단계 카드 묶음 그대로>

## 사용자 Q&A 답변
<카테고리별 정리. 답변 빈 곳은 "[답변 없음 — 정보 전달 톤으로]" 명시>

## 작성 규칙 요약
- CLAUDE.md `## 글쓰기 핵심 원칙` 4 개 (결론 먼저·한 문장 한 내용·능동형·숫자 받침) 능동 적용
- 골든 톤 표본 (조대협 블로그) 결로 따라간다
- 어색 패턴 카탈로그 능동 회피 (둘레 / ~로 넘어간 / ~을 다루다 / ~에 가깝다 / ~다는 분석이 늘었다 / ~기 때문이다 / 등)
- 인라인 출처 `(출처: URL)` — 공식 발표·원문 인용·최신성 주장만
- ILJ 키워드 본문 자연 등장 곳에 `<!--ILJ-->` 마커 (강제 끼움 X)
- `## 태그`, `## 인터럴 링크` 섹션 포함 (들어오는 링크 키워드 2 개)
- `## 이미지 메타` 블록은 제외 (3단계에서 자동 생성)
- 길이 3,150~3,850 자
- 본문 H2 직후 `![alt](images/<슬러그>/section-N.webp)` 마커 (마무리 H2 제외). H1 바로 아래 thumb 마커.

## 후크 강화 (필수)
- 한국어 사실 카드 중 가장 강한 무기 (구체 수치·갈등·인물·결과)를 H1 또는 도입 첫 단락에 끌어 올린다.
- 정의형 H1 (`~ 란?`, `~ 를 다루는 새 분야`) 회피. 결론·수치·결과를 담은 자연 문장.
- 본문 안쪽에서야 등장하는 핵심 사례를 도입에 끌어낸다.

## 자가 점검 (저장 전 통과)
1. CLAUDE.md `## 글쓰기 핵심 원칙` 4 개 다 적용했는가? (결론 먼저 / 한 문장 한 내용 / 능동형 / 숫자 받침)
2. 어색 패턴 카탈로그 (둘레 / ~로 넘어간 / ~을 다루다 / ~에 가깝다 / ~다는 분석이 늘었다 / ~기 때문이다 / A 와 B 와 C / 등) 본문에 없는가?
3. 영어 수동태 직역 (`~ 됐다`, `~ 시켜진다`, `~ 가 되어 있다`, `~ 가 보고됐다고 알려졌다`) 없는가?
4. H1·H2·H3 명사형 강제 비틀기 없는가? (자연스러우면 동사형이 더 좋다)
5. 한국어 사실 카드의 사실이 그대로 반영됐는가? (윤색·각색 X)
6. 사용자 답변에 없는 도구·가드·단계가 본문에 끼어 들어가지 않았는가?
7. 인라인 출처·ILJ 마커·이미지 마커 다 들어갔는가?
8. H1·도입·마치며에 후크가 살아 있는가? 정의·평론 톤이 아닌가?

다 통과해야 저장. 저장 후 "저장 완료 — N 자" 만 출력.
```

저장 후 사용자에게 "초안 작성 완료 (Codex), 자가 검수로 진행" 만 알린다. 본문 텍스트는 출력하지 않는다.

#### 2-B. Codex 자가 검수 1 회 (JSON 리포트)

`Agent` 툴로 새 `codex:codex-rescue` sub-agent 호출 (별도 인스턴스). 외부 글을 평가한다는 톤으로 프롬프트 작성.

```
너는 한국어 블로그 검수자다. 다음 글을 평가하고 JSON 리포트만 출력해라.

파일: posts/YYYY-MM-DD-<슬러그>.md

## 평가 룰 (CLAUDE.md 발췌)
<글쓰기 핵심 원칙·골든 톤 표본·제목 구조·도입부·문체·톤 가드·영어 용어·SEO 본문>
<blog-review SKILL.md 의 결정론 룰 10-e (어색 패턴 grep 카탈로그) + LLM 판단 17·19 본문>

## 자연 한국어 판단 기준 (사용자 메모리)
<관련 메모리 본문>

## 단계 컨텍스트 — SKIP 항목 (3단계에서 처리됨)
- 9-c 썸네일 마커 부재 / 본문 H2 직후 section-N 카드 마커 부재
- 10-c 이미지 alt 부재 / 자리채우기 / 중복
- `## 이미지 메타` YAML 블록 부재
- 추가 체크 SEO_4자리_일관 의 썸네일 alt 곳도 미평가

## 평가 항목
1. 결정론 위반 (문체·이모지·도입부·em-dash 등)
2. 어색 패턴 grep 카탈로그 (CLAUDE.md `## 톤 가드`)
3. 자연 한국어 (영어 어순 직번역·명사화 마무리·격언·평론·메타 멘트)
4. 후크 (H1·도입에 구체 수치·갈등·결과)
5. 사실 출처 의심
6. 사용자 답변에 없는 도구·가드 윤색

**무조건 N 건 이상 짚어라** (자가 confirmation bias 회피). 'OK' 한 줄 출력 금지. 가장 약한 곳 최소 3 개는 무조건 짚는다.

출력은 JSON:
{
  "score": <0-100>,
  "verdict": "PASS" | "FAIL",
  "결정론_위반": [...],
  "어색_패턴": [{"line": N, "원문": "...", "이유": "...", "대안": "..."}, ...],
  "자연_한국어": [...],
  "후크_평가": "...",
  "사실_출처_의심": [...],
  "윤색_의심": [...]
}

PASS 컷오프: score >= 90 + 결정론 위반 0 + 어색 패턴 0~1.
```

#### 2-C. Codex 재작성 1 회 (FAIL 시만)

검수가 `verdict == "FAIL"` 이면 같은 `codex:codex-rescue` sub-agent 에게 재작성 위임. 검수 JSON 을 그대로 넣고 위반 항목 모두 반영해 1 회만 다시 쓰게 한다. 재검수는 하지 않는다 (시간 통제 + 자가 confirmation 무한 루프 회피).

`verdict == "PASS"` 면 재작성 스킵.

#### 2-D. 루프 출력 요약

```
✅ 본문 단계 완료
파일: posts/YYYY-MM-DD-<슬러그>.md
- Codex 작성 완료 (N 자)
- Codex 자가 검수: <PASS | FAIL, score N/100>
- Codex 재작성: <적용 / 스킵>
주요 지적: <어색 패턴 N 건, 자연 한국어 N 건, 사실 출처 N 건>
다음 단계: 카드 메타 생성
```

### 3단계: 카드 메타 자동 생성 (메인 Claude)

`/blog-research` SKILL.md 13단계 규칙 그대로. 사용자 확인 X.

- 팔레트 3 색 (글 톤에 맞게 새로 결정, 직전 글과 안 겹치게)
- thumb 1 개 + section-N (마무리 제외 본문 H2 수만큼, 최대 4)
- 카드 타이틀은 본문 H2 메시지와 정렬
- alt 자동 작성 — 카드별 30~80 자 한국어 묘사. thumb alt 에만 핵심 키워드 자연 포함. section-N alt 는 해당 H2 메시지 중심. 카드 간 alt 중복 X
- 본문 마크다운 `![alt](path)` 의 alt 도 동기화 (Edit 일괄 갱신)

### 5단계: X 아티클 요약 (메인 Claude)

카드 렌더 완료 후, 메인 Claude 가 X(트위터) 아티클 후킹 요약 1 개를 본문 파일 끝에 넣는다.

규칙:
- 길이 100~160 자
- 후킹 톤 — 호기심 자극 + 구체 수치·갈등
- 전체 글 핵심 압축
- 평서체, 1인칭 후크 1 구절 허용
- CLAUDE.md `## 글쓰기 핵심 원칙` + 어색 패턴 카탈로그 능동 회피
- 후보 3 안 X (자동 흐름이라 1 개 확정)

저장 형식 — `## 이미지 메타` 다음:

```markdown
## X 아티클 요약

<후킹 요약 1~2 줄>
```

## 규칙

- **본문 작성·검수는 전부 Codex**. Claude 의 한국어 본문 작성·검수 sub-agent (general-purpose 의 blog-review --judge 호출 포함) 는 사용하지 않는다. 회귀 검증에서 직번역체·어색 결합 반복 확인됐다 (2026-05-15).
- **번역과 작문 분리**: 영문 Tavily → 한국어 사실 카드는 메인 Claude. Codex 는 한국어 카드만 받음. 영문 원문 노출 X.
- **컨텍스트 사전 주입**: CLAUDE.md·메모리·ILJ 표를 메인 Claude 가 발췌해 Codex 프롬프트에 넣음. cat 호출로 시간 쓰지 않음.
- **Codex 자가 검수 1 회 + 재작성 1 회**: 검수 PASS 면 재작성 스킵. FAIL 이면 재작성 1 회. 재검수는 하지 않음.
- **카드 메타·렌더·X 요약**: 메인 Claude 가 처리.
- `blog-review --fix` 자동 교정은 사용하지 않음.
- Tavily 키 (`TAVILY_API_KEY`) 가 `.env` 에 없으면 1단계에서 중단.
- **Claude 의 보강 개입 금지**: 사용자가 글이 별로라고 짚어도 Claude 가 직접 H1·도입·마치며 등을 다시 쓰지 않는다. 보강 필요 시 Codex 에게 위임.

## 시간 예상

- 2단계 (Codex 작성 + 자가 검수 + 재작성): 약 8~12 분 (사전 주입으로 cat 빠짐, 재작성 스킵 시 더 짧음)
- 3~5단계 (카드 메타·렌더·X 요약): 약 1~2 분

총 약 10~15 분.

## 에러 대응

- 1단계에서 사용자가 중단하면 1.5단계로 진행하지 않음
- 2단계 Codex 호출 실패 시 1 회 재시도, 그래도 실패하면 사용자에게 알림 후 수동 진행
- Codex plugin 미설치 환경: `Agent` 호출 실패 시 사용자에게 `/codex:setup` 안내 후 중단 (Claude 가 대신 작성하지 않는다)
- 카드 렌더 실패 시 해당 카드만 재시도: `/blog-cards <파일> --only <key>`
