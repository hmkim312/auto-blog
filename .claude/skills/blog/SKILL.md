블로그 파이프라인. 주제 탐색 → 초안 → 사용자와 H3 단위 피드백 → 가벼운 검수 → 카드 생성 순서로 진행한다.

> **권장 모델:** Sonnet 4.6. 글쓰기 품질은 Opus와 큰 차이 없고 비용·속도가 유리함. 현재 모델이 Opus면 `/model sonnet` 으로 전환 후 진행 권장. 특히 2단계(H1/H2/H3 피드백)는 반복 호출이 많아 Opus면 비용이 빠르게 쌓인다.

## 사용법

```
/blog                              # 주제 탐색부터 시작
/blog uv 패키지 매니저              # 주제 직접 지정
/blog --profile hmkim
/blog uv --profile hmkim
```

## 실행 흐름

### 1단계: 초안 생성 (blog-research 위임)

`blog-research` 스킬의 전체 흐름을 그대로 수행한다:

1. CLAUDE.md 규칙 및 `blog-research` SKILL.md 규칙을 적용
2. **주제 탐색** (`scripts/discover-topics.py` → 후보 10개 → 사용자 선택) → 주제 확정
3. 세션 ID 생성 → 프로파일 로드 → AI 쿼리 제안 → 사용자 승인 → Tavily 검색 (≤6 쿼리)
4. 심화 키워드 1~2회 (선택, 총 Tavily 호출 ≤8)
5. **검색 맞춤 Q&A** 를 한 번에 출력하고 사용자 답변 대기
6. `posts/YYYY-MM-DD-<슬러그>.md` 에 초안 + 인라인 출처 작성. **`## 이미지 메타` 블록은 이 단계에서 생성하지 않는다** — H3 피드백 완료 후 4단계에서 만든다.

### 2단계: 플로우 설명 + 피드백

**모델 체크 (필수)**: 현재 모델이 Opus면 사용자에게 "피드백 단계는 반복 호출이 많아 Sonnet 권장. `/model sonnet` 으로 바꾸시겠어요?" 안내. 사용자 결정 받고 진행.

초안 파일 경로를 밝히고, 먼저 전체 글의 구조와 의도를 설명한다:

**2-A. 전체 플로우 설명**

다음 형식으로 글 전체 흐름을 한 번에 보여준다:

```
H1: <제목> — <이 제목을 고른 이유>
H2 1: <제목> — <이 섹션이 다루는 것>
  H3: ...
  H3: ...
H2 2: <제목> — <이 섹션이 다루는 것>
  ...
마치며: <제목> — <한 줄 결론 방향>

전체 흐름: <독자가 읽으면 어떤 순서로 무엇을 알게 되는지 2~3줄>
```

이 설명을 보고 사용자가 구조 수정 요청을 하면 반영한 뒤 다음 단계로.

**2-B. H1 → H2 → H3 순서 피드백**

구조 확정 후 순서대로 진행:

1. **H1** 제목 먼저 — 제목 OK/수정 대기
2. **H2 + H3 제목** 한 섹션씩 — 제목 먼저 OK 받고 본문으로
3. **H3 본문** — 텍스트 시도안 보여주고 OK 받고 Edit 적용
4. OK면 다음으로. 수정 요청이면 다시 시도 → OK → 적용
5. 모든 섹션 완료 후 3단계로

**진행 방식:**
- 텍스트 시도안 먼저 보여주고, OK 받기 전에 파일에 쓰지 않는다
- 제목·본문 둘 다 피드백 대상
- 사용자가 직접 수정한 내용은 그대로 보존

### 3단계: 검수

피드백 완료 후 전체를 한 번에 돌려 결과 출력:

- **글자수** — 총 글자수 (3,150~3,850 기준, 부족해도 크게 문제 없음)
- **오탈자** — 발견한 목록. 없으면 "없음"
- **전체 플로우** — H1→H2→마치며 흐름 어색한 곳. 없으면 "자연스러움"
- **유입 점수** — blog-review SKILL.md 19번 5축 합산 (100점 만점), 낮은 축 보강 제안 1줄

### 4단계: 카드 메타 생성

검수 완료 후 `/blog-research` SKILL.md 13단계 규칙을 따라 `## 이미지 메타` 블록을 새로 생성:

- 팔레트 3색 (글 톤·직전 글과 겹치지 않게)
- thumb 1개 + section-N (마무리 제외 본문 H2 수만큼, 최대 4개)
- 각 카드 타이틀은 **피드백 완료된 본문 H2 내용과 정렬**
- 생성 후 사용자에게 보여주고 수정 있으면 반영

### 5단계: 카드 이미지 생성

```
uv run scripts/render-cards.py posts/<파일>
```

생성 완료 후 파일 경로 안내.

## 규칙

- 자동 검수/교정(blog-review --fix) 은 실행하지 않는다.
- H3 피드백은 텍스트 시도 → OK → Edit 적용 순서. OK 받기 전에 파일에 쓰지 않는다.
- 카드 이미지는 카드 메타 확인 후 생성.
- Tavily 키(`TAVILY_API_KEY`)가 `.env` 에 없으면 1단계에서 중단.

## 에러 대응

- 1단계에서 사용자가 중단하면 2단계로 진행하지 않음
- 카드 렌더 실패 시 해당 카드만 재시도: `/blog-cards <파일> --only <key>`
