블로그 초안 검수. CLAUDE.md 가 단일 정관이고, 이 스킬은 그 룰을 결정론 (정규식) + LLM 판단으로 적용한다.

권장 모델: Opus 4.7 (`Agent` 툴 `subagent_type: claude`, `model: opus`). 작성자도 Opus 지만 검수는 다른 컨텍스트의 sub-agent 로 분리해 자기 합리화를 깬다.

## 사용법

```
/blog-review                              # posts/ 최신 파일
/blog-review posts/2026-04-22-x.md        # 특정 파일
/blog-review <파일> --fix                 # 검수 + 안전 치환
/blog-review <파일> --judge               # /blog 자동 루프용 JSON 점수 리포트
```

## `--judge` 모드 (자동 루프용)

`/blog` 가 호출. JSON 점수 리포트만 출력. 포맷·기준은 `/blog` SKILL.md `### 2-B. Opus 검수` 절 단일 출처.

## 입력 해석

- 인자에 `.md` 경로 있으면 그 파일, 없으면 `posts/` 최신 수정 `.md`.
- `--fix` 있으면 안전 치환까지, 없으면 리포트만.

## 결정론 체크 (정규식·카운팅)

CLAUDE.md 룰을 코드 레벨로 강제. 위반 시 [FAIL], 통과 시 [OK].

1. **파일명 형식**: `^\d{4}-\d{2}-\d{2}-.+\.md$` (CLAUDE.md `## 17`)
2. **H1 정확히 1 개**
3. **H2** (꼬리 메타 `## 태그`, `## 인터럴 링크`, `## 이미지 메타`, `## X 아티클 요약` 제외):
   - 본문 3 개 (기본) 또는 4 개 (선택 섹션 포함). 5+ WARN.
   - "기술 설명", "개발 경험 공유", "오늘 배운 것" 그대로면 FAIL.
4. **H3**: 섹션당 3~4 권장 (2 이하 WARN).
5. **이모지 X** (유니코드 이모지 범위).
6. **분량** (frontmatter·코드블록·꼬리 메타·이미지 마커 제외, CLAUDE.md `## 12`):
   - 본문 2,000 자 미만이면 FAIL. 상한 강제 X.
   - 핵심 키워드 본문 자연 등장 5 회 미만이면 FAIL (CLAUDE.md `## 11`).
7. **문장 길이**: 40 자 초과 3 개 이하 OK, 초과 시 WARN.
8. **문체**: 음슴체 금지 — `~했음\.`, `~하였음\.`, `~임\.`, `~함\.`, `~였음\.`
9. **마치며**: "오늘 배운 것:" 그대로면 FAIL.
10. **가독성**:
    - H3 본문 빈 줄 없이 한 덩어리면 FAIL.
    - 한 문단 5 문장 초과 WARN.
    - 한 H3 안 문단 1 개뿐이면 WARN.
11. **도입부 두 단락** (CLAUDE.md `## 6`, H1 직후, 이미지·HTML 주석 제외):
    - 단락 2 개 아니면 FAIL.
    - 첫 단락 100~180 자 벗어나면 WARN.
    - 첫 단락에 1인칭 (`나는`, `내가`, `처음엔`, `~했었다`) FAIL.
    - 두 단락 합산 250 자 미만 WARN.
    - `## 도입부` H2 들어가면 FAIL.
    - 썸네일 마커가 H1 바로 아래 없으면 FAIL.
12. **코드 블록**: 1 개 이상 (없으면 WARN). 언어 명시 + 주석 1 줄 권장.
13. **체크리스트·비교 표**: 가독성 룰에서 금지 → 사용 시 FAIL.
14. **핵심 키워드** (CLAUDE.md `## 11`):
    - 본문 전체 5 회 미만 WARN.
    - H1·도입부·썸네일 alt 3 자리 누락 시 각각 FAIL.
15. **이미지 alt** (CLAUDE.md `## 11 이미지 alt`):
    - alt 비었거나 자리채움 (`이미지: 썸네일`, `이미지: section-N`) FAIL.
    - alt 30~80 자 벗어나면 WARN.
    - 같은 alt 두 번 이상 FAIL.
    - em-dash 부제 alt FAIL.
    - 본문 H2 직후 section-N 카드 마커 없으면 WARN.
16. **em-dash (—)**: 본문·제목 어디든 검출 시 FAIL → 쉼표·마침표·괄호로 대체.
17. **어색 어휘 grep** — `awkward-words.md` 가 단일 출처. 그 파일의 각 단어 또는 명백한 활용형 (`박는다`/`박혀`/`박아`, `잡는다`/`잡혔다` 등) 이 본문에 등장하면 해당 줄을 인용하고 맥락에 맞는 자연 한국어인지 짚는다. 대체어 추천 X — 작성자가 맥락 보고 판단. 새 어색 어휘 발견 시 `awkward-words.md` 에 추가.

## LLM 판단 항목

리포트 [판단] 섹션.

1. **입문자 눈높이**: 풀이 없는 전문 용어 5 개까지 지적.
2. **비유·예시 활용**: 개념 설명에 비유·예시 있는지.
3. **섹션 흐름**:
   - 섹션 1: 개념→필요성→동작 순서.
   - 선택 섹션: 주제 적합·중복 X.
   - 섹션 2: 1인칭 또는 함정·실전.
   - 마치며: 3 줄 이내 요약.
4. **제목 품질** (CLAUDE.md `## 7`):
   - 정의형·의문형 4 개 초과 WARN.
   - 각 H2/H3 가 섹션 내용을 드러내는 구체 명사·동사 포함.
   - 명사형 강제 X (동사형 자연스러우면 동사형 OK).
5. **톤 일관성**: 음슴체 섞이지 않고 일관.
6. **자연 한국어 종합** — grep 카탈로그가 못 잡는 결:
   - 영어 어순 직번역 (어순 단위까지).
   - 명사화 마무리 (`~ 부분`, `~ 자리`, `~ 영역`) 한 글에 3 개 이하.
   - 격언·평론 톤 (`작은 규칙이지만 누적되면 ...`, `~ 한 이름이다`).
   - 메타 멘트 (`이 글에서는 ~ 다룬다`).
   - 구어·은어 (`박힌다`, `때려넣`, `갈아탄다`).
   - 풀이 없는 외래어 (일반 독자 글 한정).
   - CLAUDE.md `## 5. 골든 톤 표본` 과 비교.
   각 위반은 문장 인용 + 대안 1 개. 5 건 이상이면 [FAIL].
7. **키워드 강박 + 카드 정렬 + 경험 사실성**:
   - 키워드가 H1·H2·H3·도입·결론·카드까지 다 박히면 본질 가려짐 → H2·H3 본질·메커니즘으로 다시 쓰기 제안.
   - 카드 타이틀이 H2 메시지와 어긋나면 지적.
   - 카드 title/subtitle 한 문장 분할 (`title:"짧을수록"`+`subtitle:"잘 따른다"`) FAIL.
   - Q&A 답변에 없는 도구·가드 끌어다 적은 흔적 윤색 지적.
   - H2-카드 적합도 (CLAUDE.md `## 15`).
8. **유입 잠재력 점수 (100 점, 90+ PASS)** — 5 축 각 20 점:
   - 키워드 매칭: H1·도입 첫 단락·썸네일 alt·파일명 4 자리 동일 키워드.
   - 신선도: 7 일 이내 또는 영구 검색 수요.
   - 제목 클릭율: 구체 수치·인물·결과·갈등.
   - 차별화: 본인 시각·경험.
   - SEO 첫 단락: 100~180 자, 정의 또는 상황 제시, 1인칭 X.

   90 미만이면 가장 낮은 축 1 개 짚어 보강 제안 1 줄.

## 출력 포맷 (리포트 모드)

```
# 블로그 리뷰 리포트
파일: posts/2026-05-15-x.md
총평: PASS / FAIL (결정론 위반 N 개, 유입 점수 N/100)

## 결정론 체크
[OK]   파일명 형식
[FAIL] 어색 어휘 — L24 "모델 둘레의 도구"
[WARN] 40 자 초과 문장: 3 개

## 분량
총 3,665 자 (≥ 2,000 → OK) · 핵심 키워드 본문 등장 7 회 → OK

## 판단 (LLM)
- 자연 한국어: L31 "~ 다는 분석이 늘었다" 직번역
- 후크: H1 정의형, 30 위→5 위 수치를 도입에 끌어 올릴 것

## 교정 제안
1. L24 "둘레의" → "환경의"
2. L31 "분석이 늘었다" → "자주 짚힌다"
```

## `--fix` 안전 치환

수정 대상 (안전):
- 음슴체 → 평서체 (`~했음.` → `~했다.`, 코드블록 제외)
- 이모지 제거
- 마지막 H2 가 화이트리스트 외면 `## 마치며` 로 교체
- "오늘 배운 것:" 제거
- em-dash → 쉼표·마침표·괄호
- 어색 어휘 1:1 치환 가능 자리 (예: "둘레의" → "환경의")

수정 금지 (LLM 판단 영역):
- 분량 조정 (제안만)
- 제목·소제목 변경
- 문장 구조 변경
- 전문 용어 풀이 추가

교정 후 diff 출력. 원본은 `<파일>.bak` 백업.

## 실행 흐름

1. 대상 파일 결정
2. 결정론 체크
3. LLM 판단
4. 리포트 출력
5. `--fix` 면 안전 치환 + 백업 + diff

## 규칙

- CLAUDE.md 충돌 시 CLAUDE.md 우선.
- frontmatter·코드블록·꼬리 메타 (`## 태그` 부터 끝까지) 분량·문체 체크 제외.
- 본문 이미지 마커도 분량 계산 제외.
- `--fix` 는 반드시 백업 후 수정.
- 어색 어휘 누적 — 새 자리 발견 시 `awkward-words.md` 에 추가.
