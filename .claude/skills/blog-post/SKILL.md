블로그 글 초안을 질의응답으로 만드는 스킬.

## 사용법

```
/blog-post <주제>
```

예: `/blog-post Docker 입문`

## 실행 흐름

### 1단계: 질의응답 (5개 질문)

주제를 받으면 아래 5개 질문을 **한 번에** 출력한다. 사용자가 답하면 2단계로 넘어간다.

```
주제: {주제}

글을 잘 쓰려면 몇 가지 물어봐야 한다.

Q1. 이 주제를 다루게 된 계기가 뭔가? (어떤 문제를 만났나, 어떤 필요가 있었나)
Q2. 기술적으로 핵심 포인트 1~3가지를 꼽는다면?
Q3. 이 주제를 다루면서 막혔거나 삽질했던 경험이 있나?
Q4. 그 문제를 어떻게 해결했나?
Q5. 이 글을 읽는 입문자에게 가장 전달하고 싶은 한 문장은?
```

### 2단계: 마크다운 초안 생성

답변을 바탕으로 CLAUDE.md의 규칙에 따라 초안을 생성한다.

**파일명**: `YYYY-MM-DD-주제-슬러그.md` (오늘 날짜 사용)  
**저장 위치**: 프로젝트 루트 `posts/` 폴더

**구조**:
```markdown
# {대제목}

## 기술 설명

### {소제목1}
### {소제목2}

## 개발 경험 공유

### {경험 소제목1}
### {경험 소제목2}

## 마치며
```

### 3단계: 이미지 프롬프트 생성

초안 하단에 아래 형식으로 이미지 프롬프트를 추가한다.
DALL-E, Midjourney, Stable Diffusion 등에 붙여넣어 사용한다.

```markdown
---
## 이미지 프롬프트 (DALL-E / Midjourney용)

**대표 썸네일**
> {주제를 시각적으로 표현하는 영문 프롬프트, flat illustration style, clean background, tech blog thumbnail}

**섹션 1 삽화**
> {기술 개념을 설명하는 다이어그램 또는 일러스트 프롬프트}

**섹션 2 삽화**
> {개발 경험/삽질 장면을 표현하는 프롬프트, developer at computer, frustrated then happy}
```

## 규칙

- CLAUDE.md의 문체, 구조, 분량 규칙을 반드시 따른다
- 이모지 사용 안 함
- 전문 용어 첫 등장 시 괄호로 설명 추가
- 코드 블록 사용 시 언어 명시 및 주석 포함
- 질문에 답이 없는 항목은 "[작성 필요]"로 표시하고 넘어간다
